## OpenCommand Center Agent System

Built a multi-agent framework with dynamic spawning and UI positioning:

- **AgentManager.js** – central registry; loads `agent_registry.json`, instantiates agent classes, handles dispatch
- **agent_registry.json** – full agent manifest: id, name, role, class, position (x,y,panel), icon, permissions, capabilities (10 agents)
- **AgentBase.js** – abstract base class with lifecycle (`initialize`, `process`), state, logging, permissions, event emit, `getStatus()`
- **ExecutiveAgent.js** – CEO-level agent; approves, escalates, sets strategy; emits events

Agents defined (10 total):
- Executive (CEO) – left panel

## Key Decisions

- Use custodial API signing (Unisat/NodesNow) to avoid private key management
- Agent UI layout: three columns (left, center, right) with explicit (x,y) coordinates
- Executive agent holds admin/override permissions
- Each agent declares capabilities; permissions drive access control
- Dispatching via `AgentManager.dispatchTask(roleOrId, payload)` matches first agent by role/id

## Web UI Implementation (Added 04:23 EDT)

- Created `index.html`: three-panel layout, modal for task input, status bar
- Created `style.css`: dark theme, agent cards, responsive grid
- Created `frontend.js`: loads `agent_registry.json`, renders cards, dispatches tasks to `/agents/:id/task`
- Created `server.js`: optional Express server to serve static files and proxy task dispatch to AgentManager
- UI shows agent icon, name, role, status (idle/active/error), and a "Send Task" button
- Task input as JSON with simple example; result displayed in modal

To run:
- Place `index.html`, `style.css`, `frontend.js`, `agent_registry.json` in same directory
- Serve with any static HTTP server (e.g., `npx serve`, `python -m http.server`, or use `server.js` with Express)
- Ensure backend `AgentManager` is running and CORS allows POST `/agents/:id/task`
